# IA Robustness Guide

Use this file when the user asks which Internet Appendix results the repo
generates, which are paper-only, or how the paper's robustness checks are
organized.

## Core Question

Which IA results are implemented in this repo, which are paper-only, and what
does the paper's broader robustness landscape look like?

## Short Answer

The repo implements 9 IA estimation models and generates a substantial subset
of IA tables and figures. The paper's full IA contains 34 tables and 50 figures
spanning 11 appendix sections. The repo covers the core posterior-probability,
pricing, treasury, sparse, and IS/OS-switch outputs. The broader
kappa-tilt, factor-exclusion, sample-split, and data-source-variation results
are paper-only.

## Paper Refs

- Section 4 (main text robustness summary)
- IA.1 through IA.11 (individual appendix sections)
- Tables IA.I through IA.XXXIV
- Figures IA.1 through IA.50

## IA Section Roadmap

The Internet Appendix is organized into 11 sections. Here is each section, what
it tests, and whether the repo generates any of its outputs.

| IA Section | Topic | Repo Status |
|------------|-------|-------------|
| IA.1 | Data construction details | Documentation only |
| IA.2 | Simulation design details for Figure 1 | Tracked fixtures; optional regeneration via `tools/run_figure1_simulation.*` |
| IA.3 | Additional empirical results (sub-model posteriors, pricing, expanding windows) | Partially implemented (Tables IA.1-7, Figure IA.17) |
| IA.4 | PEAD robustness across size terciles | Paper-only (Tables IA.XIII-XIV) |
| IA.5 | Discount-rate vs. cash-flow factor decomposition | Paper-only (Table IA.XV, Figure IA.21); main Table 5 uses the DR/CF classification |
| IA.6 | Treasury component analysis | Paper-only (Tables IA.XVI-XX); main Figures 8-9 cover the key result |
| IA.7 | Additional thousands-of-tests pricing | Paper-only |
| IA.8 | SDF dynamics, persistence, and GARCH analysis | Paper-only (Table IA.XXI); main Figures 10-11 cover the key result |
| IA.9 | Robustness to kappa tilts, imposed sparsity, factor exclusion | Sparse-joint and IS/OS-switch implemented; kappa-tilt and exclusion are paper-only |
| IA.10 | Robustness across data sources and sample periods | Paper-only (Tables IA.XXVII-XXXIII, Figures IA.39+) |
| IA.11 | CREDIT factor construction robustness | Paper-only (Table IA.XXXIV, Figure IA.50) |

## How To Answer IA Questions

1. Check if the exhibit is in `docs/manifests/exhibits.csv` (implemented subset)
2. If not, check `docs/manifests/manuscript_exhibits.csv` (full paper inventory)
3. For implemented outputs: trace to code via exhibits.csv entry_script column
4. For paper-only outputs: cite the IA section and table numbers; state clearly
   that this evidence is from the manuscript, not generated by the repo
5. Never claim a paper-only table is an executable repo output

## Common Misreadings

- IA.4 (PEAD robustness appendix section) is not the same as Table IA.4 (the
  stock no-intercept posterior probability table in the repo)
- The 9 IA models in the repo are not the same as the 9+ robustness families in
  the paper's Section 4 summary
- "Implemented-subset" means the repo generates some IA outputs, not all of them
- The sparse-joint model tests imposed sparsity (Section 4, IA.9), not a
  different dataset or sample period
